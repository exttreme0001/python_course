# Консольное приложение для управления списком студентов

Это приложение позволяет управлять списком студентов и их оценками через консольный интерфейс. Данные хранятся и загружаются из CSV-файлов.

## Цели и задачи

*   **Цель:**
    - Освоить базовые конструкции Python: условия, циклы, списковые включения.
    - Научиться проектировать архитектуру приложения: разделение на модули и пакеты.
    - Применить коллекции Python (списки, словари) для реализации бизнес-логики.
    - Реализовать собственную иерархию исключений для грамотной обработки ошибок.
    - Освоить работу с файлами формата CSV (чтение, запись, работа с заголовками).
    - Внедрить автоматическое тестирование с помощью `pytest`.

*   **Задачи:**
    1.  Спроектировать модель данных `Student` (id, ФИО, оценки).
    2.  Реализовать логику обработки данных: добавление, удаление, обновление, сортировка (по 3 критериям), расчет статистики (индивидуальной и групповой).
    3.  Реализовать модуль ввода-вывода для работы с CSV (поддержка файлов с заголовками и без, выравнивание колонок).
    4.  Создать интерактивное консольное меню (CLI) для управления приложением.
    5.  Реализовать экспорт ТОП-N лучших студентов.
    6.  Обработать краевые случаи и ошибки (дубликаты ID, отсутствие файлов, битые данные).
    7.  Покрыть код автотестами.

## Архитектура проекта

Проект структурирован как Python-пакет `lab`, что обеспечивает изоляцию пространств имен и удобство импорта.

*   `lab/`:
    *   `__init__.py`: Делает директорию пакетом.
    *   `main.py`: Точка входа. Содержит цикл `while`, обработку меню и перехват всех исключений верхнего уровня.
    *   `models.py`: Содержит класс `Student`. Реализует свойство `average` для автоматического расчета среднего балла.
    *   `io_utils.py`: Отвечает за `csv.reader` и `csv.writer`. Обрабатывает кодировку UTF-8 и параметр `newline=""`.
    *   `processing.py`: "Чистые" функции бизнес-логики. Здесь реализованы сортировки (stable sort), фильтрация, поиск Min/Max по группе.
    *   `errors.py`: Иерархия исключений (`AppError`, `StudentNotFoundError`, `DuplicateStudentIdError` и др.).
*   `data/`: Хранит файлы `students.csv`.
*   `tests/`: Содержит модульные тесты (`test_processing.py`, `test_io.py`, и т.д.) и фикстуры `pytest`.

## Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone <your-repo-url>
    cd student_manager
    ```

2.  **Установите зависимости для разработки (pytest):**
    ```bash
    pip install -r requirements-dev.txt
    ```

3.  **Запустите приложение:**
    Точка входа находится в `lab/main.py`. Для корректного запуска из корня проекта используйте флаг `-m`.
    ```bash
    python -m lab.main
    ```

## Запуск тестов

Для проверки корректности работы всех модулей запустите тесты с помощью `pytest`.
```bash
pytest -v
```
